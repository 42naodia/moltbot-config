### 🎯 关键词确定技巧

#### 基本原则
1. **核心词 + 限定词**：先确定核心概念，再加限定条件
2. **中英文都试**：技术类优先英文，生活类优先中文
3. **同义词扩展**：一个词搜不到就换同义词
4. **逐步缩小范围**：从宽泛到精确

#### 关键词组合公式
```
[核心功能] + [技术栈/平台] + [限定条件]
示例：
- 知乎热榜 + 爬虫 + Python
- zhihu + hot list + crawler + python
- 小红书 + spider + 无需登录
```

#### 常见搜索词变体
| 中文 | 英文变体 |
|------|----------|
| 爬虫 | crawler, spider, scraper |
| 热榜/热搜 | hot list, trending, hot search |
| 接口/API | api, interface |
| 下载 | download, downloader |
| 聚合 | aggregator, all-in-one |

#### 搜索词演变策略
**四个演变方向**：

| 方向 | 说明 | 示例 |
|------|------|------|
| **向上抽象** | 放大领域，找更通用的方案 | 知乎爬虫 → 热榜聚合 → 数据采集 |
| **向下具体** | 聚焦细节，找更精准的方案 | 热榜爬虫 → 知乎热榜API → 知乎热榜无登录 |
| **横向扩展** | 找相关概念，发现新思路 | 爬虫 → 聚合服务 → 第三方API |
| **同义替换** | 换个说法，覆盖更多结果 | crawler → spider → scraper |

**关键洞察**：
- 搜不到时不要死磕，换个方向
- 有时候「绕路」反而更快到达目的地
- 聚合方案往往比单点方案更有价值

### ✅ 实践要求（每次必须调用）

1. **遇到任何搜索任务**，先调用这个框架
2. **先判断领域和平台**，再执行搜索
3. **应用对应的优劣标准**进行筛选
4. **搜不到时**，使用搜索词演变策略调整
5. **每次回答前**，主动标注信息局限性和不确定性
6. **定期回顾**，持续优化分析方法和框架

### 📌 执行示例

**场景**：需要找知乎热点采集的Python方案

```
1. 判断领域：技术/代码类
2. 确定平台：GitHub
3. 了解标准：star数 + 更新时间
4. 执行搜索：
   - 关键词：zhihu hot crawler python
   - 排序：sort:stars
   - 筛选：最近1年有更新的
5. 对比方案：
   - 方案A：500 stars，3个月前更新
   - 方案B：200 stars，1周前更新
   - 选择：优先B（更活跃）或A（更成熟）视情况而定
```
